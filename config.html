1 Installation

Clone into the .emacs.d folder created by Emacs 

git clone https://github.com/blemberger-b/emacs-config.git ~/.emacs.d

When Emacs is opened, it will download the specified packages (with their dependencies) and apply the customizations from init.el and this file.

2 Custom Packages & Modes

These packages here and elsewhere are installed via use-package (https://github.com/jwiegley/use-package)

2.0.1 Install & use sudo-edit (from MELPA)

sudo-edit allows editing write protected files by first doing sudo before visting the file 

(use-package sudo-edit
  :ensure t
  :bind ("s-e" . sudo-edit))

2.0.2 Dashboard

The dashboard package allows setting a better initial buffer when Emacs opens. See https://github.com/emacs-dashboard/emacs-dashboard for info and configuration values. Also, here we are excluding files from the recent files list that get treated as edited by the package manager for some reason. 

(use-package dashboard
  :ensure t
  :pin melpa-stable
  :config
  (dashboard-setup-startup-hook)
  (setq dashboard-items '((recents . 10)))
  (setq dashboard-banner-logo-title
        "The more I learn about people, the more I like my dog\n\t\t\t\t\t\t\t\t\t--Mark Twain")
  (setq dashboard-set-footer nil)
  (setq recentf-exclude '(".*autoloads\.el" ".*\.gz")))

2.0.3 Install & use markdown-mode (from MELPA)

This package provides a major mode for editing markdown files. It requires the markdown command be installed via homebrew. 

(use-package markdown-mode
  :ensure t
  :commands (markdown-mode gfm-mode)
  :mode (("README\\.md\\'" . gfm-mode)
         ("\\.md\\'" . markdown-mode)
         ("\\.markdown\\'" . markdown-mode))
  :init (setq markdown-command "/usr/local/bin/markdown"))

2.0.4 Presentation Mode

This minor mode allows increasing/decreasing the font size in all buffers. It can be run on-demand: M-x presentation-mode. See https://github.com/zonuexe/emacs-presentation-mode. 

(use-package presentation
  :ensure t
  :pin melpa-stable)

3 Global Settings

3.0.1 Install & run exec-path-from-shell

On OS X, an Emacs instance started from the graphical user interface will have a different environment than a shell in a terminal window, because OS X does not run a shell during the login. Obviously this will lead to unexpected results when calling external utilities like make from Emacs. This library works around this problem by copying important environment variables from the user's shell. See https://github.com/purcell/exec-path-from-shell 

(use-package exec-path-from-shell
  :ensure t
  :init (when (memq window-system '(mac ns x))
          (exec-path-from-shell-initialize)))

3.0.2 Turn off emacs start screen

(setq inhibit-startup-message t)

3.0.3 Turn off aggressive vertical scrolling

(setq scroll-conservatively 100)

3.0.4 Turn off annoying bell

(setq ring-bell-function 'ignore)

3.0.5 Turn off auto-saving

(setq auto-save-default nil)

3.0.6 Turn off stupid auto-backups

(setq make-backup-files nil)

3.0.7 Set default fill column

Set the default column to 100 where emacs will attempt to auto-fill in those modes that support it. 

(setq-default fill-column 100)

3.0.8 Cursor blink

Set the cursor to always blink. This setting has no effect in TTY mode (the cursor always blinks there). 

(setq blink-cursor-blinks 0)

3.0.9 Replace 'yes-or-no' prompts everywhere with 'y-or-n' prompts

(defalias 'yes-or-no-p 'y-or-n-p)

3.0.10 Remap Mac Fn key to super key (s- )

Doesn't work in tty mode, must use C-x @ s and then the key. 

(setq mac-function-modifier 'super)

3.0.11 Fix some stupid failed binding to backtab key in Mac OS console mode

See https://stackoverflow.com/questions/3518846/shift-tab-produces-cryptic-error-in-emacs. 

(add-hook 'term-setup-hook
          (lambda () (define-key input-decode-map "\e[Z" [backtab])))

3.0.12 iterm2 Mac OS console mode fixes

Emacs generally has problems with the Control-Meta-Shift combination key bindings in console mode. This is due to terminal emulators on all systems, not just Mac OS, not able to process the escape sequences and pass them through to emacs. Some of these problems can be fixed by modifying the terminal emulator program to send the expected escape sequences for the desired key-presses that emacs is expecting. A writeup for how to fix some of these key bindings in Mac OS iterm2 is discussed here: https://emacs.stackexchange.com/a/45624. The decimal ASCII codes to be used for whatever bindings are desired can be looked up in any standard ASCII table.

For example, to map C-, in iterm2 so that it works when emacs is run in console mode, I configured my iterm2 profile to map ^â€‹, to the escape sequence [44;5u since comma is ASCII code 44.

4.0.1 Set geometry of all emacs frames

Top-left corner of initial frame will be 50 pixels down from the top margin and 250 pixels right from the left margin. It also makes the window size 45 x 112 characters.

(use-package smex

     ("M-x" . smex))

5.0.2 Add which-key package

This package provides a pop-up menu of command key completions when keys are pressed interactively in the minibuffer. We also suppress it from showing up in the modeline. See https://github.com/justbur/emacs-which-key.

(use-package diminish

  :ensure t
  :config (diminish 'eldoc-mode))

14 Line Manipulation

14.0.1 Copy whole line

Copy the line containing the point to the kill ring. 

(defun copy-whole-line ()
  (interactive)
  (save-excursion
    (kill-new
     (buffer-substring
      (point-at-bol)
      (point-at-eol)))))

Bind it to a key: 

(global-set-key (kbd "C-c l") 'copy-whole-line)

15 Buffers

15.0.1 Bind ibuffer to the normal buffer list key

ibuffer is a better interactive buffer list that allows deleting buffers and switching buffers in the same window. 

(global-set-key (kbd "C-x C-b") 'ibuffer)

15.0.2 Buffer Move

Package buffer-move allows re-arranging buffers within multiple windows in a frame. The keybindings below conflict with org-mode, but they are too damn convenient, so just use them when not in org-mode. "<C-S->" here means Ctrl+Shift+whatever. 

(use-package buffer-move
  :ensure t
  :pin melpa-stable
  :bind (("<C-S-up>" . buf-move-up)
         ("<C-S-down>" . buf-move-down)
         ("<C-S-right>" . buf-move-right)
         ("<C-S-left>" . buf-move-left)))

15.0.3 Always kill the current buffer

Override default emacs and ido-mode functionality to prompt for the buffer to kill, and just go ahead and kill the current one. It will still prompt for confirmation if the buffer is unsaved. 

(defun kill-current-buffer ()
  (interactive)
  (kill-buffer (current-buffer)))

Bind it to the default kill-buffer keybinding C-x k. 

(global-set-key (kbd "C-x k") 'kill-current-buffer)

15.0.4 Kill all buffers

This command will kill all killable buffers. If any buffers have unsaved changes, you'll be prompted to confirm. 

(defun kill-all-buffers ()
  (interactive)
  (mapc 'kill-buffer (buffer-list)))

Bind it to some difficult key combination to press by accident: C-M-s-k 

(global-set-key (kbd "C-M-s-k") 'kill-all-buffers)

16 Windows

16.0.1 Split window horizontally & put point in new window

(defun split-and-follow-horizontally ()
  (interactive)
  (split-window-below)
  (balance-windows)
  (other-window 1))

Bind to the default split horizontally key C-x 2. 

(global-set-key (kbd "C-x 2") 'split-and-follow-horizontally)

16.0.2 Split window vertically & put point in new window

(defun split-and-follow-vertically ()
  (interactive)
  (split-window-right)
  (balance-windows)
  (other-window 1))

Bind to the default split vertically key C-x 3. 

(global-set-key (kbd "C-x 3") 'split-and-follow-vertically)

16.0.3 Halve other window height

Tip given at https://stackoverflow.com/questions/4987760/how-to-change-size-of-split-screen-emacs-windows. 

(defun halve-other-window-height ()
  "Expand current window to use half of the other window's lines."
  (interactive)
  (enlarge-window (/ (window-height (next-window)) 2)))

Bind it to a key. 

(global-set-key (kbd "C-c v") 'halve-other-window-height)

16.0.4 Bind more convenient keys to resize windows

Note: These override the default bindings for scroll window up/down, etc., but I don't care because I use C-v, M-v for that. The arrow keys on Mac keyboards are given in comments next to each definition below. 

(global-set-key (kbd "<next>") 'shrink-window) ; Fn-down-arrow
(global-set-key (kbd "<prior>") 'enlarge-window) ; Fn-up-arrow
(global-set-key (kbd "<home>") 'shrink-window-horizontally) ; Fn-left-arrow
(global-set-key (kbd "<end>") 'enlarge-window-horizontally)  ; Fn-rigt-arrow

17 Text Mode

17.0.1 text-mode

In text mode, you generally want to auto-wrap or "auto-fill" lines of text, so this makes that happen but only in text-mode and its derivatives. We also enable flyspell-mode minor mode in all text modes to get spellchecking functionality, and we bind M-s to the go to character function in the avy package (because we don't want to use avy in non-text modes). 

(add-hook 'text-mode-hook (lambda () (turn-on-auto-fill)
                            (flyspell-mode)
                            (local-set-key (kbd "M-s") 'avy-goto-char)))
  1. Note: This requires that the ispell program be installed in the path of the local system.

18 Programming Modes

18.0.1 prog-mode hook

The parent major mode for all programming language modes is prog-mode. Here we add the subword-mode minor mode to this mode, which will allow navigation by word commands to respect camel-case words. In other words, camel-case boundaries of identifiers will form "word" boundaries in all programming language files with this setting enabled. Also, turn on spell-checking in comments and literal strings using flyspell-prog-mode (This requires that the ispell program be installed in the path of the local system). 

(add-hook 'prog-mode-hook (lambda ()
                            (subword-mode 1)
                            (diminish 'subword-mode)
                            (flyspell-prog-mode)
                            (diminish 'flyspell-mode)))

18.0.2 Auto-complete

For auto-complete in programming modes, we'll use the company package (https://company-mode.github.io/). 

(use-package company
  :ensure t
  :pin gnu
  :hook (('prog-mode . company-mode)
         ('cider-repl-mode . company-mode)))

18.0.3 Turn on parenthesis matching mode

(show-paren-mode 1)

18.0.4 Paredit

This minor mode makes dealing with parenthesis and lisp expressions much easier. Here, we enable it for some lisp major modes only. Also remap the s-expression slurp and barf commands to keybindings that work in both GUI and TTY. 

(use-package paredit
  :ensure t
  :pin melpa-stable
  :hook ((clojure-mode
          emacs-lisp-mode
          scheme-mode
          geiser-repl-mode
          cider-repl-mode) . enable-paredit-mode)
  :bind (("C-c )" . paredit-forward-slurp-sexp)
         ("C-c (" . paredit-backward-slurp-sexp)
         ("C-c M-)" . paredit-forward-barf-sexp)
         ("C-c M-(" . paredit-backward-barf-sexp)))

18.0.5 Colorful matching delimiters

This package changes the color of corresponding pairs of parenthesis and brackets. Here, we enable it only in prog-mode and its descendants, and in cider-repl-mode. 

(use-package rainbow-delimiters
  :ensure t
  :pin melpa-stable
  :hook (('prog-mode . rainbow-delimiters-mode-enable)
         ('cider-repl-mode . rainbow-delimiters-mode-enable))
  )

19 Clojure

In addition to the above settings, the following settings apply only to editing clojure files and buffers.

19.0.1 Set indent mode for clojure buffers

Set the variable clojure-indent-style to the value always-indent. See https://github.com/clojure-emacs/clojure-mode#indentation-options for more info. 

(setq clojure-indent-style 'always-indent)

19.0.2 clojure-mode customizations

  • comment-add only use one ; when creating block clojure comments
  • turn on eldoc-mode (for some reason, this is necessary to get ELDoc to work in clojure buffers)
(add-hook 'clojure-mode-hook (lambda () (setq-local comment-add 0)
                               (turn-on-eldoc-mode)))

19.0.3 Install & use cider (from MELPA)

cider is the clojure integrated development environment (https://github.com/clojure-emacs/cider). We're only going to use the melpa stable version. Initialize some settings, and bind the command to clear the REPL screen to C-c C-l key. Also set the following defaults:

  • cider-repl-display-help-banner suppress the verbose help banner when opened
  • cider-eldoc-display-context-dependent-info try to auto-complete function calls with in-scope parameters
  • cider-allow-jack-in-without-project Suppress warning that the REPL gives when you're current buffer is not part of a leiningen project.
  • nrepl-log-messages Log all request/response messages between cider and the nrepl server to a special buffer for debugging purposes
  • cider-use-overlays controls whether evaluated results are shown in the original buffer. Here, we disable in TTY mode, because the fonts & colors are messy. Results will be on the status line.
  • cider-prompt-for-symbol Here, we set this variable to only prompt for the symbol if the lookup for the symbol at point fails.
  • cider-repl-pop-to-buffer-on-connect Display the REPL buffer on connect, but don't move focus to it.
(use-package cider
  :ensure t
  :pin melpa-stable
  :init (setq cider-repl-display-help-banner nil)
  (setq cider-eldoc-display-context-dependent-info t)
  (setq cider-allow-jack-in-without-project t)
  (setq nrepl-log-messages t)
  (setq cider-repl-result-prefix "=> ")
  (unless window-system (setq cider-use-overlays nil))
  (setq cider-prompt-for-symbol nil)
  (setq cider-repl-pop-to-buffer-on-connect 'display-only)
  :bind ("C-c C-l" . cider-repl-clear-buffer))

19.0.4 cider-repl-mode

Major mode for the Cider REPL. Enable subword mode in it, but diminish it. 

(add-hook 'cider-repl-mode-hook (lambda ()
                            (subword-mode 1)
                            (diminish 'subword-mode)))

20 Scheme

20.0.1 Install & use geiser

geiser is a better scheme mode than the built-in scheme-mode. Specify guile command as the name of the scheme REPL (https://www.gnu.org/software/guile/manual/html_node/index.html#SEC_Contents). guile can be installed via homebrew or downloaded otherwise. The guile command must be included in the PATH.

Start a scheme REPL by running the command M-x run-geiser. 

(use-package geiser
  :ensure t
  :init (setq geiser-active-implementations '(guile))
  (setq geiser-repl-query-on-exit-p nil))
